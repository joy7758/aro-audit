# Contributing

## Contribution License
By submitting a contribution (code, docs, tests, examples), you agree that:
- You have the legal right to submit the contribution.
- The contribution is your original work, or you have proper permission.
- Your contribution is licensed under the repository LICENSE (MIT).

## IP and Copyright Requirements
Please do not submit:
- Proprietary code copied from employers or clients without written permission.
- Source files with incompatible license terms.
- Content that infringes copyright, trademark, or trade secret rights.

## Source Headers
For new source files, include SPDX and copyright header where practical:
- `SPDX-License-Identifier: MIT`
- `Copyright (c) 2026 joy7758 contributors`

## Commit Sign-Off (Recommended)
Use sign-off to assert origin and licensing intent:

```bash
git commit -s -m "your message"
```

## Security and Compliance
Do not commit secrets, private keys, credentials, or regulated data.
Use redacted samples for demos and tests.
