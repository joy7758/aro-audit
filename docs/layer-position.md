# Layer Position

ARO-Audit is the Audit Evidence Layer in the five-layer Digital Biosphere Architecture.

## Why evidence is not the same as execution

Execution answers what happened during runtime. Evidence answers what can be exported, reviewed, replayed, and independently checked after that runtime has completed. These are adjacent concerns, but they are not the same object class.

## Why evidence is not the same as interaction

Interaction objects describe the requested intent, the proposed action, and the returned result semantics. Audit evidence describes the reviewable post-execution record. Conflating them makes it harder to distinguish requests, execution, and receipts.

## Minimal handoff model

interaction object -> execution trace -> audit receipt
